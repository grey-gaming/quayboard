# Architecture Decision Records

ADRs capture decisions that refine, constrain, or override the project outline.

## When To Write One

Add an ADR when a change affects:

- repository structure
- framework or tooling choices
- schema or API contract strategy
- testing strategy
- design-system rules
- workflow, governance, or review policy

## How To Use This Directory

- Copy `000-template.md` to the next available three-digit number.
- Use a short, stable slug in the filename.
- Link relevant ADRs from `README.md`, milestone docs, or implementation docs when they affect active work.
- Update or supersede prior ADRs instead of leaving contradictions unresolved.

## Status Values

Use one of these statuses:

- `proposed`
- `accepted`
- `superseded`
- `rejected`
